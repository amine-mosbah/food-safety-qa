# Food Safety Q&A Generator (fsqa)

Synthetic Q&A generation and evaluation for **SemEval-2025 Task 9** food recall incidents.

A 4-stage LLM pipeline: question generation, required/optional item extraction, answer generation, and LLM-as-judge scoring.

> **v0.3** — Refactored into a CLI (`fsqa`) and an n8n form-driven workflow. Both read the **same official CSV** (`data/food_recall_incidents.csv`, 7,546 records) and let the user pick the cluster at runtime.

---

## What the pipeline does

For each food recall record in a chosen cluster:

```
Stage 1 ── Question Generation     (LLM, temperature 0.7)
            ↳ question + one-sentence justification

Stage 2 ── Required + Optional      (LLM, temperature 0.0)
            Answer Items
            ↳ atomic facts the answer MUST / SHOULD contain

Stage 3 ── Answer Generation        (LLM, temperature 0.0)
            ↳ gold answer, grounded in source recall only

Stage 4 ── LLM-as-Judge             (LLM, temperature 0.0)
            ↳ required_coverage, optional_coverage, overall_score, verdict
```

Output → one row per Q&A pair in Excel (CLI) or Google Sheets (n8n).

## Two ways to run it

### Option A — Python CLI (recommended)

```bash
# 1. Install
pip install -r requirements.txt

# 2. One-time: store your OpenAI key (chmod 600, ~/.config/fsqa/openai-key)
./fsqa setup

# 3. Explore the dataset
./fsqa explore                              # by hazard-category (default)
./fsqa explore --by product-category --top 10
./fsqa explore --by country --top 20

# 4. Sample a cluster
./fsqa sample --by hazard-category --value allergens --n 50

# 5. Run the 4-stage pipeline
./fsqa run --input samples/allergens_50.csv --out outputs/allergens.xlsx

# OR — single interactive command that walks you through everything
./fsqa wizard
```

### Option B — n8n workflow (visual, browser-based)

1. Import `n8n/food-safety-qa-pipeline.json` into your n8n instance (top-right `⋮` → Import from File).
2. Add an **OpenAI** credential to the 4 OpenAI nodes.
3. Add a **Google Sheets** credential to the `Append to Google Sheet` node and create a destination sheet with a `qa_pairs` tab.
4. Open the form URL (n8n exposes it when the workflow is active), then:
   - Upload `data/food_recall_incidents.csv`
   - Pick a hazard category
   - Enter sample size, Sheet ID, tab name
   - Submit → the pipeline runs and writes to your sheet.

## CLI commands

| Command | What it does |
|---|---|
| `fsqa setup` | Stores your OpenAI key locally (`~/.config/fsqa/openai-key`, chmod 600). |
| `fsqa explore [--by COL] [--top N]` | Prints a table of the dataset broken down by a cluster dimension. `--by` accepts `hazard-category` (default), `product-category`, `country`, `year`. |
| `fsqa sample --by COL --value V --n N [--out PATH]` | Stratified-samples N records from the chosen cluster (stratifies on `hazard` by default). Writes a CSV with stable `record_id`s. |
| `fsqa run --input CSV --out XLSX` | Runs the 4 LLM stages on every row of the sample. Live progress + cost tracker. |
| `fsqa wizard` | End-to-end interactive walkthrough: explore → pick cluster → sample → confirm cost → run. |

All commands accept `--input PATH` to override the default `data/food_recall_incidents.csv`.

## v0.2 → v0.3 changes

- ✅ **One source-of-truth CSV.** The old train/valid/test splits and pre-sliced `allergens_*.csv` files are gone. The pipeline now reads `data/food_recall_incidents.csv` (7,546 records) and slices clusters at runtime.
- ✅ **Single CLI (`fsqa`)** replaces the two old scripts (`sample_cluster.py`, `run_pipeline.py`). Sub-commands, rich progress UI, interactive wizard.
- ✅ **n8n workflow is now form-driven.** Upload your CSV, pick a cluster from a dropdown, type your Sheet ID — no JSON edits.
- ✅ **Output writes to Google Sheets** instead of being baked into the workflow as a binary.
- 🔄 Prompts unchanged from v0.2 (including the ECHO KEY ENTITIES rule that lifted req_coverage from 0.89 → 0.96).
